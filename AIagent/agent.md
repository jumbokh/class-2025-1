

## **一、AI 基本概念**

* **Prompt（提示詞）**

  * User Prompt：使用者輸入的訊息
  * System Prompt：定義 AI 的角色、性格、背景
  * 個性化提示：結合使用者需求與系統提示，產生更貼近情境的回應

---

## **二、AI 代理（Agent）**

* AI 回答問題有限，任務執行仍需人類操作
* **AutoGPT 範例**：協助管理電腦檔案
* **Agent 定義**：

  * 作為 **AI 模型** 與 **使用者** 之間的橋樑
  * 能調用各種 **代理工具（Agent Tools）** 來完成任務

---

## **三、功能呼叫（Function Calling）**

* 問題：AI 回應格式可能不一致
* 解決方案：Function Calling

  * 統一工具描述與 AI 輸出格式
  * 提高開發與訓練效率
* 限制：目前不同公司 API 標準仍未統一

---

## **四、MCP 協議（Model Context Protocol）**

* **作用**：標準化 AI 代理與工具服務的互動
* **結構**：

  * MCP 伺服器：提供工具、資源、提示模板（可在本地或網路運行）
  * MCP 客戶端：負責與模型交互，傳遞資源
* **兩種溝通方式**：

  1. 系統提示（System Prompt）
  2. 函數調用（Function Calling）

---

## **五、AI 協作架構與未來展望**

* **代理運作流程**：

  1. 從 MCP 伺服器檢索工具
  2. 轉換為系統提示 / 函數調用格式
  3. 與使用者提示一同傳給模型
  4. 調用工具（如 `web_browse`）並整合結果
  5. 生成回應給使用者
* **整體架構**：如齒輪般協同運作（User Prompt + System Prompt + Agent + MCP + 工具）
* **未來展望**：

  * AI 技術快速進步帶來挑戰與焦慮
  * 鼓勵積極學習，主動理解並擁抱 AI 變革
